# Plan 10-01: Recreate DB Views with cliente_id and Aliases

## Objective

Fix the broken `v_turnos_semana` and `v_deudas_proximas` Supabase views so they return `cliente_id` and correct aliases (`nombre_cliente`, `estado_visa`), enabling all dashboard links.

## Context

The current views omit `cliente_id` and use raw column names (`nombre`, `estado`) instead of the aliases the TypeScript interfaces in `app/(dashboard)/page.tsx` expect (`nombre_cliente`, `estado_visa`). This causes every link in the dashboard Turnos and Deudas tables to resolve to `href="#"` because `t.cliente_id` and `d.cliente_id` are `undefined`. The fix is entirely in the DB -- no code changes are needed since the TS interfaces already declare the correct field names.

Call the Supabase MCP tool `mcp__5b08abd5-390d-44ab-a57f-0164c7a57b58__apply_migration` with:

- **project_id**: `ixonzhjorzjcyyujzgel`
- **name**: `fix_views_cliente_id_aliases`
- **SQL** (both statements in one migration):

```sql
CREATE OR REPLACE VIEW v_turnos_semana AS
  SELECT
    v.visa_id,
    v.cliente_id,
    c.gj_id,
    c.nombre AS nombre_cliente,
    c.telefono,
    v.fecha_turno,
    v.estado AS estado_visa
  FROM visas v
  JOIN clientes c ON c.id = v.cliente_id
  WHERE v.fecha_turno >= CURRENT_DATE
    AND v.fecha_turno <= (CURRENT_DATE + '7 days'::interval)
  ORDER BY v.fecha_turno;

CREATE OR REPLACE VIEW v_deudas_proximas AS
  SELECT
    p.pago_id,
    p.cliente_id,
    c.gj_id,
    c.nombre AS nombre_cliente,
    c.telefono,
    p.monto,
    p.fecha_vencimiento_deuda,
    p.fecha_vencimiento_deuda - CURRENT_DATE AS dias_restantes
  FROM pagos p
  JOIN clientes c ON c.id = p.cliente_id
  WHERE p.estado = 'DEUDA'::estado_pago
    AND p.fecha_vencimiento_deuda IS NOT NULL
    AND p.fecha_vencimiento_deuda <= (CURRENT_DATE + '30 days'::interval)
  ORDER BY p.fecha_vencimiento_deuda;
```

This is a single migration with both `CREATE OR REPLACE VIEW` statements separated by a semicolon.
