## Objective

Permitir que desde /tramites el admin pueda registrar un cliente nuevo y su visa en un solo modal, sin necesidad de ir primero a /clientes.

## Context

### Archivos de referencia
- `components/visas/IniciarVisaModal.tsx` — patrón base de modal de visa (no modificar)
- `components/clientes/NuevoClienteModal.tsx` — patrón de formulario de cliente (no modificar)
- `app/api/clientes/route.ts` — POST crea cliente (estado ACTIVO auto), GET lista clientes
- `app/api/visas/route.ts` — POST crea visa con cliente_id

### Decisiones de diseño (del CONTEXT)
- Dos modos: "Nuevo cliente" (default) y "Cliente existente"
- Toggle entre modos con dos tabs/botones en el header del modal
- Modo nuevo cliente: nombre*, telefono*, canal*, estado visa, notas
- Modo cliente existente: buscar en lista (input filter client-side), estado visa, notas
- Submit nuevo cliente: POST /api/clientes → con id resultante → POST /api/visas
- Submit cliente existente: POST /api/visas directamente
- Error 409 duplicado: mostrar card de duplicado (igual que NuevoClienteModal)
- No incluir todos los campos de cliente (email, dni, etc.) — solo los mínimos requeridos

## Tasks

### Task 1: Crear NuevoTramiteModal

Crear `components/visas/NuevoTramiteModal.tsx` como client component.

**Props:**
```tsx
interface Props {
  open: boolean
  onOpenChange: (open: boolean) => void
  onSuccess: () => void
}
```

**States:**
- `modo: 'nuevo' | 'existente'` — default 'nuevo'
- Modo nuevo: `nombre, telefono, canal` (strings)
- Modo existente: `clienteId: string`, `clientes: ClienteOption[]`, `busqueda: string`
- Visa: `estadoVisa: EstadoVisa` ('EN_PROCESO' default), `notas: string`
- Control: `loading, serverError, saved, errors, clienteDuplicado`

**Modo 'nuevo' — submit flow:**
1. Validar nombre, telefono, canal requeridos + estadoVisa
2. POST /api/clientes: `{ nombre, telefono, canal }` (sin estado — server pone ACTIVO)
3. Si 409 → mostrar clienteDuplicado card
4. Si ok → tomar `data.id` del cliente creado
5. POST /api/visas: `{ cliente_id, estado: estadoVisa, notas }`
6. Si ok → setSaved(true), onSuccess(), cerrar tras 1200ms

**Modo 'existente' — submit flow:**
1. Validar que clienteId esté seleccionado
2. POST /api/visas: `{ cliente_id: clienteId, estado: estadoVisa, notas }`
3. Si ok → setSaved(true), onSuccess(), cerrar

**Fetch clientes (modo existente):**
- Al cambiar a modo 'existente': fetch GET /api/clientes
- Guardar en `clientes` state como `{ id, nombre, gj_id }[]`
- Input de búsqueda filtra client-side por `nombre.toLowerCase().includes(busqueda)`
- Mostrar como `<select>` filtrado o lista con radio buttons — usar `<select>` simple

**Estructura JSX:**
```
Dialog
  DialogHeader: "Nuevo trámite de visa"
  ── Toggle de modo (2 botones: Nuevo cliente / Cliente existente)
  ── Body:
      [Si modo === 'nuevo']
        Nombre*, Teléfono*, Canal* (grid 2 cols)
        Estado visa, Notas
        [Si clienteDuplicado] → card duplicado
      [Si modo === 'existente']
        Input búsqueda + Select con clientes filtrados
        Estado visa, Notas
  ── Footer: Cancelar | Crear trámite
  ── Overlay éxito (igual que IniciarVisaModal)
```

**Estilos:** copiar `inputStyle`, `labelStyle` de IniciarVisaModal.

### Task 2: Integrar en tramites/page.tsx

`app/(dashboard)/tramites/page.tsx` es un server component — no puede tener state directamente.

**Solución:** Convertir la sección del header a un client wrapper o pasar el botón a TramitesTable.

**Opción elegida:** Agregar el botón directamente en `TramitesTable` (ya es client component). Pasar `onNuevoTramite?: () => void` no es necesario — el modal maneja su propio open state.

**Mejor opción:** Crear un pequeño client wrapper `NuevoTramiteButton` en el mismo archivo o como componente separado, e importarlo en la page. Esto mantiene la page como server component.

Patrón: crear en `components/visas/NuevoTramiteModal.tsx` un export adicional `NuevoTramiteButton` que contiene el state de open y el botón trigger.

```tsx
export function NuevoTramiteButton() {
  const router = useRouter()
  const [open, setOpen] = useState(false)
  return (
    <>
      <button onClick={() => setOpen(true)}>+ Nuevo trámite</button>
      <NuevoTramiteModal
        open={open}
        onOpenChange={setOpen}
        onSuccess={() => router.refresh()}
      />
    </>
  )
}
```

En `tramites/page.tsx`: importar `NuevoTramiteButton` y renderizarlo en el header junto al título.
