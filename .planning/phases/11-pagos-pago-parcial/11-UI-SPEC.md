## Interaction Contract

### Desglose panel — display conditions

```
Show desglose grid when ALL of:
  - resolverDeuda === true
  - deudaDelTipo exists (deudas.find(d => d.tipo === tipo))
  - estado === 'PAGADO'
  - monto input is non-empty AND parseFloat(monto) > 0

Hide desglose grid when monto is empty string (not when 0) — avoid confusion while typing
```

Source: CONTEXT.md D-02, D-03 + Specifics section

### Desglose grid layout

```
3 equal columns inside the amber panel:
  Col 1 — "Total"    value: deudaDelTipo.monto formatted
  Col 2 — "Pagado"   value: monto ingresado formatted
  Col 3 — "Resto"    value: (deudaDelTipo.monto - montoNum) formatted

CSS: display grid, grid-template-columns: 1fr 1fr 1fr
Column label: text-xs text-gj-secondary
Column value: text-[13px] text-gj-text font-semibold

Formatting: formatPesos() from lib/utils.ts (consistent with app-wide pattern)
```

Source: CONTEXT.md D-03, code_context

### "Archivar deuda restante" toggle

```
Show toggle when: resto > 0 (monto < deudaDelTipo.monto)
Hide toggle when: monto >= deudaDelTipo.monto (show green indicator instead)

Default state: ON (checked=true) when toggle first appears — prevents accidental data loss
Pattern: <input type="checkbox"> with accent-gj-amber, same as "Cancelar deuda" checkbox
Label: "Archivar deuda restante ($X)" — $X updates in real time using formatPesos(resto)
Label alignment: flex items-center gap-2 cursor-pointer
```

Source: CONTEXT.md D-04, D-05, D-06, D-07

### "Pago completo" indicator

```
Show when: monto >= deudaDelTipo.monto AND resolverDeuda === true AND estado === 'PAGADO'
Content: "Pago completo — cancela la deuda"
Styling: text-gj-green font-semibold text-xs
Position: replaces desglose grid + toggle (those are hidden)
```

Source: CONTEXT.md D-08, D-09

### Fade-in transition (Claude's discretion)

```
Apply CSS transition when desglose appears/disappears:
  transition-opacity duration-150 ease-in-out
  opacity-0 → opacity-100 when resto becomes > 0
Use conditional className approach, not JS animation library
```

Source: CONTEXT.md "Claude's Discretion" — animations delegated here

### Submit behavior with archivarResto

```
When archivarResto === true AND resto > 0:
  1. PATCH /api/pagos/${deudaDelTipo.id} → { estado: 'PAGADO', monto: montoNum, fecha_pago }
  2. POST /api/pagos → { cliente_id, tipo, monto: resto, fecha_pago, estado: 'PENDIENTE' }
     — NO fecha_vencimiento_deuda (intentional — keeps pago out of v_deudas_proximas)
     — If tipo === 'VISA', include visa_id in both calls

When archivarResto === false OR resto === 0:
  Normal PATCH flow (existing behavior, unchanged)
```

Source: CONTEXT.md D-12, D-13
